## Программное средство управления подарочными сертификатами

Проект на базе `Laravel` + `Inertia.js (Vue 3)` + `Tailwind CSS` для внутреннего учета и управления подарочными сертификатами.

### Основные возможности

- **CRUD сертификатов**: создание, просмотр, редактирование и удаление подарочных сертификатов.
- **Генерация кода**: при создании сертификата автоматически генерируется уникальный код.
- **Баланс и номинал**: хранится номинал и текущий баланс по каждому сертификату.
- **Статусы сертификатов**: черновик, активен, погашен, истёк, отменён.
- **Срок действия**: опциональное поле `expires_at` с проверкой при погашении.
- **Получатель**: имя и email получателя, поле заметок для внутренних комментариев.
- **История операций**: отдельная таблица транзакций с типами `issue`, `redeem`, `adjust`.
- **Красивый интерфейс**: все экраны построены на Inertia + Vue 3 с Tailwind.

### Технологический стек

- **Backend**: Laravel (PHP)
- **Frontend**: Inertia.js + Vue 3
- **Стили**: Tailwind CSS

### Установка и запуск

1. **Установить зависимости PHP**:

```bash
composer install
```

2. **Установить JS-зависимости**:

```bash
npm install
```

3. **Сконфигурировать окружение**:

```bash
cp .env.example .env
php artisan key:generate
```

В файле `.env` укажите настройки подключения к базе данных (`DB_*`).

4. **Запустить миграции и сиды**:

```bash
php artisan migrate --seed
```

Будет создан пользователь `test@example.com` и несколько демонстрационных сертификатов.

5. **Запуск приложения**:

В одном терминале:

```bash
php artisan serve
```

В другом терминале:

```bash
npm run dev
```

Откройте приложение в браузере по адресу `http://127.0.0.1:8000`.

### Работа с сертификатами

- После входа под пользователем (`test@example.com`, пароль задайте в сидере или через сброс пароля) в верхнем меню доступен раздел **«Сертификаты»**.
- На странице списка можно:
  - Просматривать все сертификаты, их номинал, баланс, статус и срок действия.
  - Фильтровать по статусу и искать по коду, имени или email получателя.
- На странице создания:
  - Указать номинал, валюту, данные получателя, срок действия и заметки.
  - После сохранения автоматически генерируется уникальный код, статус становится `active`.
- На странице редактирования:
  - Обновлять основные поля сертификата.
  - Погашать часть или всю сумму по сертификату (баланс уменьшается, в историю записывается транзакция).
