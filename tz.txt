ТЕХНИЧЕСКОЕ ЗАДАНИЕ
SaaS-платформа управления подарочными сертификатами "CertiMarket"
Дата: 14 марта 2026 г.
Срок MVP: 10 недель
Архитектура: 2 микросервиса (Gateway + Core)
Стек: PHP 8.2 (Laravel 11), Vue.js 3, MySQL 8.0, RabbitMQ

1. БИЗНЕС-ЛОГИКА (что делаем)
Маркетплейс подарочных сертификатов для малого бизнеса:

text
Кафе "Вкусняшка" → создаёт сертификат 2000₽ → привязывает адрес "ул. Козлова 12"
Клиент покупает → получает PDF на email → может разделить с друзьями (SmartShare)
В кафе сканируют QR → списывают сумму
1.1. Ключевые атрибуты сертификата
text
✅ Сегмент: HoReCa, Розница, Услуги, Развлечения, Образование
✅ Локация: адрес + координаты (только здесь можно потратить)
✅ Номинал: фиксированный или произвольная сумма
✅ SmartShare: можно разделить между друзьями (опционально)
✅ PDF: отправляется только после покупки

4.4. ОПЛАТА И PDF
text
После покупки (всегда):
1. ЮKassa/CloudPayments
2. Платформа берёт комиссию (1.5-3%)
3. PDF отправляется на email покупателя

PDF содержит:
- QR-код + PIN
- Номинал + баланс
- Адрес локации
- Условия + срок
4.5. ГАШЕНИЕ (без PDF!)
text
Онлайн:
1. Официант сканирует QR (html5-qrcode)
2. Вводит сумму заказа
3. Баланс обновляется → чек на экране

Оффлайн:
1. Клиент называет PIN
2. Бизнес проверяет в кабинете
3. Синхронизация при интернете
5. БАЗА ДАННЫХ (MySQL)
sql
-- Сегменты
CREATE TABLE segments (
    id INT PRIMARY KEY,
    name VARCHAR(50),      -- HoReCa, Розница
    color VARCHAR(7)       -- #FF6B35
);
-- Бизнесы
CREATE TABLE businesses (
    id INT PRIMARY KEY,
    user_id INT,
    name VARCHAR(100),
    inn VARCHAR(20),
    subscription ENUM('free','start','pro'),
    verified_at TIMESTAMP
);

-- Локации
CREATE TABLE locations (
    id INT PRIMARY KEY,
    business_id INT,
    address TEXT,
    lat DECIMAL(10,8),
    lng DECIMAL(11,8),
    radius_km INT DEFAULT 5
);

-- Сертификаты
CREATE TABLE certificates (
    id VARCHAR(20) PRIMARY KEY,  -- CERT-001
    business_id INT,
    segment_id INT,
    location_id INT,
    nominal DECIMAL(10,2),
    balance DECIMAL(10,2),
    expires_at DATE,
    status ENUM('active','used','expired')
);

-- SmartShare (при делении)
CREATE TABLE certificate_splits (
    id INT PRIMARY KEY,
    parent_id VARCHAR(20),
    recipient_email VARCHAR(100),
    amount DECIMAL(10,2),
    balance DECIMAL(10,2),
    qr_path VARCHAR(255),
    pin_hash VARCHAR(255),
    created_at TIMESTAMP
);
